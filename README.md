History of version control system...
-> Local VCS
* Database - To keep track of files.
pros:- can track files and rollback
cons:- if you loose your HarDdisk, everything is lost.

-> Centralized VCS
* Data store on centralized server.
pros:- we can pulled or pushed files.
       multiple user can do this.
cons:- if main server crash we only takes the file which was latest commit we don't rollback.

-> Distributed VCS
* In this also data can store on centralized server.
pros:- we can pulled or pushed files.
       multiple user can collaborate.
       all user have complete backup of their files and projects.
